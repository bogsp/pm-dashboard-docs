# Kanban Board

The Kanban Board is the main workspace for task management. It provides a visual, drag-and-drop interface for organizing and moving tasks through your workflow.

## Overview

The Kanban Board displays tasks as cards organized in vertical columns. Users can drag cards between columns to update task status, creating an intuitive workflow visualization.

## Basic Usage
```tsx
import { KanbanBoard } from 'pm-dashboard'

export default function TaskBoard() {
  const columns = [
    {
      id: 'todo',
      title: 'To Do',
      tasks: [
        { id: '1', title: 'Design homepage', status: 'todo' },
        { id: '2', title: 'Write documentation', status: 'todo' },
      ]
    },
    {
      id: 'in-progress',
      title: 'In Progress',
      tasks: [
        { id: '3', title: 'Implement API', status: 'in-progress' },
      ]
    },
    {
      id: 'done',
      title: 'Done',
      tasks: [
        { id: '4', title: 'Setup project', status: 'done' },
      ]
    },
  ]

  return <KanbanBoard columns={columns} />
}
```

## Drag and Drop

The Kanban Board uses HTML5 drag-and-drop API for smooth task movement.

### Implementing Drag Handlers
```tsx
import { useState } from 'react'
import { KanbanBoard } from 'pm-dashboard'

export default function Board() {
  const [columns, setColumns] = useState(initialColumns)

  const handleTaskMove = (taskId, fromColumnId, toColumnId) => {
    setColumns(prevColumns => {
      // Find task in source column
      const fromColumn = prevColumns.find(c => c.id === fromColumnId)
      const task = fromColumn.tasks.find(t => t.id === taskId)
      
      // Remove from source
      const updatedFromColumn = {
        ...fromColumn,
        tasks: fromColumn.tasks.filter(t => t.id !== taskId)
      }
      
      // Add to destination
      const toColumn = prevColumns.find(c => c.id === toColumnId)
      const updatedToColumn = {
        ...toColumn,
        tasks: [...toColumn.tasks, { ...task, status: toColumnId }]
      }
      
      // Update state
      return prevColumns.map(col => {
        if (col.id === fromColumnId) return updatedFromColumn
        if (col.id === toColumnId) return updatedToColumn
        return col
      })
    })
    
    // Update backend
    updateTaskStatus(taskId, toColumnId)
  }

  return (
    <KanbanBoard 
      columns={columns}
      onTaskMove={handleTaskMove}
    />
  )
}
```

## Column Management

### Adding Columns
```tsx
const addColumn = () => {
  const newColumn = {
    id: `column-${Date.now()}`,
    title: 'New Column',
    tasks: [],
  }
  setColumns([...columns, newColumn])
}

<KanbanBoard 
  columns={columns}
  onAddColumn={addColumn}
/>
```

### Removing Columns
```tsx
const removeColumn = (columnId) => {
  setColumns(columns.filter(col => col.id !== columnId))
}
```

### Reordering Columns
```tsx
const moveColumn = (fromIndex, toIndex) => {
  const newColumns = [...columns]
  const [removed] = newColumns.splice(fromIndex, 1)
  newColumns.splice(toIndex, 0, removed)
  setColumns(newColumns)
}
```

## WIP Limits

Work-in-progress limits help prevent bottlenecks.

### Setting Limits
```tsx
const columns = [
  {
    id: 'in-progress',
    title: 'In Progress',
    tasks: tasks,
    limit: 3, // Maximum 3 tasks
  }
]
```

### Enforcing Limits
```tsx
const handleTaskMove = (taskId, fromCol, toCol) => {
  const targetColumn = columns.find(c => c.id === toCol)
  
  if (targetColumn.limit && targetColumn.tasks.length >= targetColumn.limit) {
    alert(`Column "${targetColumn.title}" is at capacity (${targetColumn.limit} tasks)`)
    return
  }
  
  // Proceed with move
  moveTask(taskId, fromCol, toCol)
}
```

### Optimistic Updates
```tsx
const handleTaskMove = async (taskId, from, to) => {
  // Update UI immediately
  updateLocalState(taskId, to)
  
  try {
    // Update backend
    await updateTaskStatus(taskId, to)
  } catch (error) {
    // Rollback on error
    updateLocalState(taskId, from)
    showError('Failed to move task')
  }
}
```

## Best Practices

### Column Design

- Keep 3-6 columns for clarity
- Use clear, action-oriented names
- Set WIP limits to prevent bottlenecks
- Color-code columns consistently

### Task Cards

- Keep titles concise (< 60 characters)
- Show essential metadata only
- Use icons for quick recognition
- Indicate priority visually

### User Experience

- Provide immediate feedback on drag
- Show loading states during updates
- Allow undo for accidental moves
- Enable keyboard navigation

## Troubleshooting

### Drag not working

Check that:
- Browser supports HTML5 drag-and-drop
- Event handlers are properly bound
- Cards have draggable attribute
- No CSS conflicts with pointer-events

### Performance issues

Solutions:
- Implement virtualization for large lists
- Use React.memo for task cards
- Debounce drag events
- Optimize re-render logic
