# Task Cards

Task Cards are the building blocks of the Kanban Board. Each card represents a single task with its status, title, metadata, and actions.

## Overview

Task Cards display essential task information in a compact, scannable format. They support drag-and-drop, show comment/file counts, and provide quick access to task details.

## Basic Usage
```tsx
import { TaskCard } from 'pm-dashboard'

export default function MyBoard() {
  const task = {
    id: '1',
    title: 'Design homepage mockup',
    status: 'in-progress',
    priority: 'high',
    assignee: { 
      name: 'Sarah Chen', 
      avatar: '/avatars/sarah.jpg' 
    },
    comments: 3,
    attachments: 2,
    dueDate: new Date('2025-01-30'),
  }

  return <TaskCard task={task} />
}
```

## Props
```tsx
interface TaskCardProps {
  task: Task
  onClick?: (task: Task) => void
  onStatusChange?: (taskId: string, newStatus: string) => void
  onDelete?: (taskId: string) => void
  draggable?: boolean
  showAssignee?: boolean
  showDueDate?: boolean
  showPriority?: boolean
  compact?: boolean
  className?: string
}

interface Task {
  id: string
  title: string
  description?: string
  status: TaskStatus
  priority?: TaskPriority
  assignee?: User
  dueDate?: Date
  createdAt: Date
  updatedAt: Date
  comments?: number
  attachments?: number
  labels?: Label[]
}

type TaskStatus = 'todo' | 'in-progress' | 'done' | 'blocked'
type TaskPriority = 'low' | 'medium' | 'high' | 'urgent'
```

### task

**Type:** `Task`  
**Required:** Yes

The task object to display.
```tsx
const task = {
  id: '1',
  title: 'Implement user authentication',
  status: 'in-progress',
  priority: 'high',
  assignee: user,
  dueDate: new Date('2025-01-25'),
  comments: 5,
  attachments: 1,
}

<TaskCard task={task} />
```

### onClick

**Type:** `(task: Task) => void`  
**Required:** No

Callback when card is clicked.
```tsx
<TaskCard 
  task={task}
  onClick={(task) => {
    openTaskModal(task)
  }}
/>
```

### onStatusChange

**Type:** `(taskId: string, newStatus: string) => void`  
**Required:** No

Callback when task status changes.
```tsx
<TaskCard 
  task={task}
  onStatusChange={(id, status) => {
    updateTask(id, { status })
  }}
/>
```

### onDelete

**Type:** `(taskId: string) => void`  
**Required:** No

Callback when delete action is triggered.
```tsx
<TaskCard 
  task={task}
  onDelete={(id) => {
    if (confirm('Delete this task?')) {
      deleteTask(id)
    }
  }}
/>
```

## Due Dates

### Date Display
```tsx
import { formatDistanceToNow, isPast, isToday, isTomorrow } from 'date-fns'

const formatDueDate = (date: Date) => {
  if (isToday(date)) return 'Today'
  if (isTomorrow(date)) return 'Tomorrow'
  if (isPast(date)) return `Overdue by ${formatDistanceToNow(date)}`
  return formatDistanceToNow(date, { addSuffix: true })
}

<TaskCard 
  task={task}
  renderDueDate={(date) => (
    <span className={isPast(date) ? 'text-red-500' : 'text-gray-600'}>
      ğŸ“… {formatDueDate(date)}
    </span>
  )}
/>
```

### Overdue Indicator
```tsx
const isOverdue = (task: Task) => {
  return task.dueDate && 
         isPast(task.dueDate) && 
         task.status !== 'done'
}

<TaskCard 
  task={task}
  className={isOverdue(task) ? 'border-red-500 border-2' : ''}
/>
```

## Best Practices

### Card Content

- Keep titles under 60 characters
- Show only essential metadata
- Use icons for quick recognition
- Maintain consistent spacing

### User Experience

- Provide immediate visual feedback
- Support keyboard navigation
- Show loading states
- Enable quick actions on hover

### Performance

- Memoize cards to prevent unnecessary re-renders
- Virtualize long lists
- Lazy load card details
- Optimize drag-and-drop

## Troubleshooting

### Cards not draggable

Check:
- `draggable` prop is true
- Drag event handlers attached
- No CSS `pointer-events: none`
- Parent container allows dragging

### Metadata not displaying

Verify:
- Data properties exist on task object
- Show* props are true
- No CSS hiding elements
- Component imports correct
