# Progress Chart

The Progress Chart is the **#1 priority widget** in PM Dashboard. It provides instant visual feedback on project health and task distribution.

## Overview

The Progress Chart displays task distribution across different statuses using horizontal bars. It's always visible at the top of the dashboard for at-a-glance project assessment.

## Basic Usage
```tsx
import { ProgressChart } from 'pm-dashboard'

export default function Dashboard() {
  return (
    <ProgressChart 
      data={[
        { label: 'To Do', value: 5, color: '#e5e7eb' },
        { label: 'In Progress', value: 3, color: '#3b82f6' },
        { label: 'Done', value: 12, color: '#10b981' },
      ]}
    />
  )
}
```

## Props
```tsx
interface ProgressChartProps {
  data: ChartData[]
  labels?: string[]
  showPercentages?: boolean
  animate?: boolean
  height?: number
  className?: string
}

interface ChartData {
  label: string
  value: number
  color: string
}
```

### data

**Type:** `ChartData[]`  
**Required:** Yes

Array of data points to display. Each item represents a status category.
```tsx
const data = [
  { label: 'To Do', value: 8, color: '#e5e7eb' },
  { label: 'In Progress', value: 4, color: '#3b82f6' },
  { label: 'Done', value: 15, color: '#10b981' },
  { label: 'Blocked', value: 2, color: '#ef4444' },
]
```

### labels

**Type:** `string[]`  
**Required:** No  
**Default:** Extracted from data

Custom labels for the chart. If not provided, uses labels from data.
```tsx
<ProgressChart 
  data={data}
  labels={['Todo', 'Active', 'Complete']}
/>
```

### showPercentages

**Type:** `boolean`  
**Required:** No  
**Default:** `true`

Display percentage values alongside counts.
```tsx
<ProgressChart 
  data={data}
  showPercentages={true}  // Shows "12 (60%)"
/>
```

### animate

**Type:** `boolean`  
**Required:** No  
**Default:** `true`

Enable smooth animations when chart updates.
```tsx
<ProgressChart 
  data={data}
  animate={true}
/>
```

### height

**Type:** `number`  
**Required:** No  
**Default:** `200`

Chart height in pixels.
```tsx
<ProgressChart 
  data={data}
  height={300}
/>
```

### className

**Type:** `string`  
**Required:** No

Additional CSS classes for styling.
```tsx
<ProgressChart 
  data={data}
  className="shadow-lg rounded-lg"
/>
```

## Dynamic Data

Update the chart with real-time task data:
```tsx
import { useState, useEffect } from 'react'
import { ProgressChart } from 'pm-dashboard'

export default function LiveChart() {
  const [tasks, setTasks] = useState([])

  useEffect(() => {
    // Fetch tasks
    fetchTasks().then(setTasks)
    
    // Subscribe to updates
    const unsubscribe = subscribeToTasks((updatedTasks) => {
      setTasks(updatedTasks)
    })
    
    return unsubscribe
  }, [])

  const chartData = [
    { 
      label: 'To Do', 
      value: tasks.filter(t => t.status === 'todo').length,
      color: '#e5e7eb' 
    },
    { 
      label: 'In Progress', 
      value: tasks.filter(t => t.status === 'in-progress').length,
      color: '#3b82f6' 
    },
    { 
      label: 'Done', 
      value: tasks.filter(t => t.status === 'done').length,
      color: '#10b981' 
    },
  ]

  return <ProgressChart data={chartData} />
}
```

## Performance

### Optimization Tips

- **Memoize data**: Prevent unnecessary recalculations
- **Debounce updates**: Batch rapid changes
- **Lazy load**: Load chart only when visible
```tsx
import { useMemo } from 'react'

const chartData = useMemo(() => {
  return calculateChartData(tasks)
}, [tasks])

<ProgressChart data={chartData} />
```

### Large Datasets

For projects with hundreds of tasks:
```tsx
// Aggregate data before charting
const aggregateData = (tasks: Task[]) => {
  const statusCounts = tasks.reduce((acc, task) => {
    acc[task.status] = (acc[task.status] || 0) + 1
    return acc
  }, {} as Record<string, number>)

  return Object.entries(statusCounts).map(([status, count]) => ({
    label: status,
    value: count,
    color: getStatusColor(status),
  }))
}
```

## Best Practices

### Data Accuracy

- Update chart data in real-time
- Validate data before rendering
- Handle zero values gracefully
- Show loading states during data fetch

### Visual Clarity

- Use distinct colors for each status
- Maintain consistent color scheme across dashboard
- Keep bar widths proportional to values
- Display clear labels and counts

### User Experience

- Position chart at top of dashboard (highest priority)
- Make chart interactive (clickable bars for filtering)
- Show percentage alongside counts
- Animate transitions for smooth updates

## Troubleshooting

### Chart not rendering

Check that:
- Data array is not empty
- All data items have required properties
- Color values are valid CSS colors
- Component is properly imported

### Incorrect percentages

Ensure total is calculated correctly:
```tsx
const total = data.reduce((sum, item) => sum + item.value, 0)
const withPercentages = data.map(item => ({
  ...item,
  percentage: Math.round((item.value / total) * 100)
}))
```
